SOCKS5Tunnel

Provides SOCKS5-capable transport services for proxied network requests.  This protocol is also used by Tor to transport various network requests.  Since TorControl is used to manage the Tor services process, if this process is already correctly configured and running SOCKS5Tunnel can be used completely independently (TorControl may be entirely omitted).

loadHTTP

public function loadHTTP(request: URLRequest):Boolean

Sends a HTTP request through the socks proxy, sending any included information (such as form data) in the process.  Additional requests via this tunnel connection will be disallowed until this one has completed (since replies may be multi-part).

Parameters

requestThe URLRequest object holding the necessary information for the request. 

Returns

True if the request was dispatched successfully, false otherwise.

establishNewCircuit

public function establishNewCircuit():Boolean

Attempts to establish a new Tor circuit through a running TorControl instance.  Future SOCKS5Tunnel instances will communicate through the new circuit while existing and connected instances will continue to communicate through their existing circuits until closed.  A TorControl instance must be instantiated and fully initialized before attempting to invoke this command.

Returns

True if TorControl is active and could be invoked to establish a new circuit, false if the invocation failed for any reason.

Invoked to authenticate the SOCKS5 tunnel after it is connected.  The tunnel will not accept any outbound requests until authentication has been completed.

Note: Currently only the 0 (none) authentication type is supported.

onTunnelData

private function onTunnelData(eventObj: ProgressEvent):void

Handles raw HTTP/HTTPS data from the SOCKS5 tunnel socket.  Authentication, tunnel establishment, and message parsing branching (how responses are interpreted), are all handled automatically in this method.

Parameters

param eventObj A standard ProgressEvent event (usually from an active Socket instance).
